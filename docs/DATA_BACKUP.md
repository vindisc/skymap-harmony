# 数据备份与恢复

## 第一期能力

“我的 > 数据保护”提供两项能力：

- **备份全部复盘**：把当前全部非删除复盘、复盘 ID、导出状态和常用设置保存为
  `.jybackup` 文件。文件由用户选择位置，位于应用沙箱之外，卸载应用后仍保留。
- **从备份恢复**：先校验文件类型、版本、SHA-256、记录数量和重复 ID，再经确认整体
  替换当前复盘库。RDB 写入使用单一事务，任何一条失败都会回滚。

常用设置包括复盘人、桌面卡片背景样式和删除动画开关。第一期不包含原图、待复盘照片、
家庭存储地址与密码；密码不得进入明文备份。应用检测到首次空库启动时会提示选择备份，
也可以稍后从“我的”页操作。

## 手工卸载恢复验收

使用非日常数据的测试设备，按顺序执行：

1. 创建至少 3 条复盘，包含已导出和未导出记录，并修改复盘人和卡片背景设置。
2. 进入“我的 > 数据保护 > 备份全部复盘”，把 `.jybackup` 保存到“下载”或云盘目录。
3. 确认成功提示中的记录数与复盘库一致。
4. 手动卸载应用，确认系统文件管理中的 `.jybackup` 仍存在。
5. 安装正常完整 HAP 并启动，在首次空库引导中选择该文件。
6. 确认恢复摘要后执行恢复，检查复盘总数、标题、判断、项目归属、导出状态和常用设置。
7. 再导出一份备份，重复恢复并确认结果稳定，没有重复数据。

## 异常验收

- 选择普通 JSON 或其他文件，应提示不是见遇完整备份。
- 修改备份正文任意字符，应提示完整性校验失败，当前数据保持不变。
- 导入未来版本或超过 32 MB 的文件，应明确拒绝。
- 在恢复确认框点击取消，当前数据保持不变。
- 自动化 `testScenario` 启动不得弹出首次恢复引导。

## 测试与发布边界

`bash scripts/test_app.sh` 会运行备份恢复契约测试和 ArkTS 构建。
`bash scripts/test_app.sh --device` 使用覆盖安装，不清理现有数据。
`--ui-test` 会清空测试安装的数据，因此只允许在专用设备执行；完成或失败后脚本会重新安装
正常完整 HAP。系统文件选择器、手动卸载和外部文件保留仍需按上面的真机流程验收。
