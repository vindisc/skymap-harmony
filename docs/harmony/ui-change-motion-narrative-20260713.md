# 叙事化动效升级变更单

## 变更等级

P2 产品变化。为主链路增加统一动效语义、用户强度设置和里程碑反馈，不调整页面信息架构与业务数据链路。

## 问题

- Tab、首页、编辑和导出菜单主要依赖整块淡入或硬切，页面之间缺少方向与层次。
- 加载占位和统计数字在状态变化时直接切换。
- 完成待复盘任务与普通保存使用同一反馈强度。

## 范围

- 新增 Motion Token、动效强度持久化和 8 个原生 ArkUI 动效组件。
- 接入 AppShell、首页、复盘库、编辑页、预览页、统计页和“显示与动效”设置页。
- 完成待复盘任务时播放一次有限时长的里程碑动画。

## 不改

- 不改路由语义、Review/Pending 数据模型、导出格式和持久化结构。
- 不改 `build-profile.json5`，不替换系统 Dialog/Toast，不引入第三方动画库。
- 不改既有 ShatterOverlay 删除语义。

## 验收标准

- 完整、舒缓、最少三档可以保存并立即影响曲线、时长、分批和庆祝动画。
- 快速切换 Tab 或关闭 Bottom Sheet 时动画可反向/取消，不堆积旧定时任务。
- 只有“待复盘 -> 已完成”首次保存触发完成仪式，重复保存不触发。
- Shimmer 离屏停止，Ceremony 同时最多一个且总时长不超过 Token 上限。
- 动效静态门禁通过，ArkTS 编译和 HAP 打包通过。

## 验收结论

代码与自动门禁通过后接受；真机流畅度、共享 Hero 连贯性和键盘态作为本轮剩余人工验收项。
