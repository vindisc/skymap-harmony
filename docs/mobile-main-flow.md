# skymap-harmony 手机端主流程

本文只描述当前 HarmonyOS 端已经落地的主流程。

## 1. 首页双入口

首页当前不是单入口，而是并行双入口：

```text
导入照片，开始复盘 -> 编辑 -> 保存进入阅读 -> 导出 / 回看
导入待复盘 -> 写入待复盘列表 -> 复盘库打开待复盘照片 -> 编辑 -> 保存进入阅读
```

### 入口 A：导入照片，开始复盘

1. 首页点击 `导入照片，开始复盘`
2. 系统相册单选一张照片
3. 成功读取尺寸后创建草稿并进入编辑页
4. 如果尺寸读取失败，保留 URI，用兜底宽高继续流程

### 入口 B：导入待复盘

1. 首页点击 `导入待复盘`
2. 系统相册多选照片，当前上限 30 张
3. 尺寸读取失败的项不会写入待复盘列表
4. 成功项写入 `pending_review_photos`
5. 用户后续从复盘库 `待复盘` 筛选进入单张编辑

当前已经支持多选待复盘导入，不再使用“待复盘不支持批量导入”的旧说法。

## 2. 编辑与保存

编辑页当前固定字段：

- 标题
- 视觉落点
- 落点原因
- 视线路径
- 画面事实
- 核心关系
- 是否成立
- 延伸理解
- 当前卡点

保存链路：

1. 编辑页点击 `保存并进入阅读`
2. 文档写入本地 RDB `reviews`
3. 同时写入 `review_exchange` 沙箱备份
4. 如果当前草稿来自待复盘照片，对应 `Pending` 状态会被更新为 `reviewed`
5. 跳转到阅读页

## 3. 阅读与导出

阅读页负责展示已保存复盘，不承担编辑态布局。

当前导出菜单包括：

- 导出 JPG 复盘卡
- 导出 `review.json`
- 导出复盘包
- 导出复盘包（含原图）
- 复制复盘数据

### JPG 导出

- 导出图来自专用导出组件，不是当前手机页面截图
- 先写应用沙箱，再通过系统保存流程进入相册
- 横图、竖图、方图共用当前 `LongFormExportReviewCard` 导出链路，但导出版式会按图片方向调整

### `review.json` 导出

- 先写沙箱备份
- 再通过系统文件保存流程交给用户选择位置
- `review.json` 只表达复盘字段，不写 bundle 路径、导出图路径或原图二进制

### 复盘包导出

- 导出到家庭存储配置目标
- 失败时给出用户可理解的配置 / 连通性反馈
- 复盘包属于外部导出资产，不参与本地复盘库主索引查询

## 4. 复盘库

复盘库当前统一入口为 `复盘库` Tab。

固定筛选：

- 全部
- 待复盘
- 成立
- 待判断
- 不成立

语义边界：

- `待复盘` 视图展示 `PendingReviewPhoto`
- 其余筛选展示已保存 `Review`
- 当前没有把待复盘任务混入已保存复盘列表模型

支持能力：

- 搜索标题、关系、卡点、文件名
- 删除待复盘照片
- 删除已保存复盘
- 从待复盘照片继续进入编辑
- 从已保存复盘进入阅读

## 5. 统计

统计页正式文案已统一为：

- `学习进度`
- `复盘结果`

### 学习进度

数据来源同时读取：

- `PendingReviewPhotoStore`
- `ReviewCardHistoryService`

当前口径：

- `待复盘 = pendingCount`
- `已完成 = 已保存复盘总数`
- `累计导入 = pendingCount + 已保存复盘总数`
- `完成率 = 已完成 / 累计导入`

### 复盘结果

只统计已保存复盘，不统计待复盘照片：

- 成立
- 待判断
- 不成立
- 最近 30 天
- 最近卡点

## 6. 当前存储边界

- 本地复盘库主索引是 RDB `reviews`
- `pending_review_photos` 单独保存待复盘照片
- `ReviewCardStore` 只保存当前编辑中的工作副本
- `review_exchange/*.review.json` 是备份 / 交换 / 有限恢复来源
- `Preferences(review_card_history.items)` 只保留旧版本迁移作用

## 7. 当前不做

- 自动同步
- 双向同步
- 冲突合并
- 账号系统
- 云端服务
- AI 自动分析
- 把待复盘任务直接当成已完成复盘结果
