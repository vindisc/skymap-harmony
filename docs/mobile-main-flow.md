# skymap-harmony v0.2 手机端主流程

`skymap-harmony` v0.2 的目标是让鸿蒙端从“能演示”进入“日常可用”。手机端只服务一条稳定链路：

```text
首页 -> 开始复盘 -> 选择照片 -> 写复盘 -> 保存进入阅读 -> 导出 review.json / 复制复盘数据 / 导出图片 -> 回首页刷新 -> 我的摄影复盘回看历史
```

## 当前主链路

1. 首页点击「开始复盘」。
2. 通过系统相册选择一张照片。
3. 编辑页填写标题与复盘字段。
4. 点击「保存并进入阅读」，记录写入本地历史。
5. 阅读页展示复盘记录，可编辑、导出 `review.json`、复制复盘数据、导出图片。
6. 回到首页时重新加载本地历史，概览、最近一次复盘、我的项目同步刷新。
7. 进入「我的摄影复盘」查看历史记录，列表支持左滑删除。

## 复盘字段

手机端 v0.2 保留固定复盘结构，不新增字段名：

- 标题
- 视觉落点
- 落点原因
- 视线路径
- 画面事实
- 核心关系
- 是否成立
- 延伸理解
- 当前卡点

空字段在阅读页和导出图中不展示；复制 JSON 时仍保留 key，值为空字符串。

## 首页

首页固定结构：

1. 摄影复盘卡
2. 副标题
3. 开始复盘按钮
4. 复盘概览
5. 最近一次复盘
6. 我的项目

首页不展示完整记录列表，不展示横图 / 竖图 / 尺寸信息。

## 首页统计口径

首页所有模块都从 `ReviewCardHistoryService.load` 返回的历史记录生成：

- 复盘概览统计全项目记录。
- 最近一次复盘取最新一条历史记录。
- 我的项目使用默认项目 `default` 的同一份历史记录。
- 当前只有 `default` 项目时，首页总数等于默认项目全部记录数。
- 成立、待判断、不成立统一通过 `normalizeReviewJudgement` 归一化。
- `成立 + 待判断 + 不成立 = 总复盘`。
- 首页可见统计必须直接绑定标量状态，不通过参数化统计项 builder 或嵌套 summary 对象渲染，避免刷新缓存导致数字不显示。
- 连续天数计算失败时只回退连续字段，不能清零总复盘、成立、待判断等可见统计。
- 历史读取失败时保留上一轮已显示统计，并通过提示文案告知用户重新进入首页或重启应用。

保存、导出标记和删除后，相关页面重新读取或重建这份历史数据；首页每次显示时都会重新加载。

相关回归记录见 [首页统计回归审计 2026-06](./home-stats-regression-audit-2026-06.md)。

## 阅读页

阅读页是「复盘记录详情」，不是导出编辑页。

固定内容：

- 顶部标题：复盘记录
- 操作入口：编辑、导出 `review.json`、复制复盘数据、导出
- 正文：照片、标题、复盘时间、成立状态、已填写字段

阅读页不显示：

- 已保存 / 已导出这类状态正文
- 已复制复盘数据正文
- 横图 / 竖图 / 尺寸信息
- 导出按钮或系统状态栏出现在导出图中

## 导出图片规则

导出目标是一张复盘长图，不是当前手机页面截图。

导出图包含：

- 照片
- 标题
- 复盘时间，位于标题下方左侧
- 成立状态，位于同一行右侧
- 视觉落点
- 落点原因
- 视线路径
- 画面事实
- 核心关系
- 是否成立
- 延伸理解
- 当前卡点

空字段不导出，不留空白。导出图不包含状态栏、返回、编辑、复制复盘数据、导出按钮、已保存、Toast 文案。v0.2 继续使用统一长图结构，不重新区分横图、竖图、方图导出版式。

导出时页面会临时挂载隐藏的导出组件并对组件快照，不对阅读页截图；系统保存弹窗出现时，背景仍保持正常阅读页。

## 复制 JSON 协议

阅读页「复制复盘数据」复制 `ReviewCardExchangeSchema v1` 的 pretty print JSON。协议字段见 [review-card-exchange-schema.md](./review-card-exchange-schema.md)。

关键约束：

- `reviewStructure` 固定为 `quickReview`。
- `decision` 输出英文枚举：`works`、`uncertain`、`notWorks`。
- `reviewerText` 来自复盘设置。
- 旧记录缺字段时不丢 key。
- 页面只显示 Toast：已复制复盘数据。

## review.json 手动交换

- 阅读页支持导出 `review.json`。
- 当前导出目录为应用文件目录 `files/review_exchange/`。
- 导出成功后，页面和 Toast 会明确显示导出路径。
- 这一步只完成手动文件交换，不代表 SMB / WebDAV 自动同步已经完成。

## 我的与设置

v0.2 只新增一个轻量设置项：复盘人。

- 我的页提供「复盘人」入口。
- 复盘人页填写复盘人名称。
- 我的页提供「同步中心」和「家庭存储」入口。
- 使用本地 preferences 持久化。
- 未设置时 `reviewerText` 输出空字符串。
- 复制 JSON 时读取当前保存的复盘人。

## 我的摄影复盘

项目详情页是长期回看页：

- 不提供显式返回按钮，依赖系统左滑返回。
- 不显示「添加复盘」大按钮。
- 列表支持左滑删除。
- 列表滚动到底部不回弹。
- 卡片缩略图统一 1:1 cover。
- 摘要优先级：核心关系、当前卡点、延伸理解、还没有提炼核心关系。
- 状态角标位于卡片右上角并弱化显示。

## 当前不做

v0.2 不做：

- 多项目创建
- 云同步
- 账号登录
- AI 分析
- Mac 端改动
- icon 改动
- 字段名称重做
- 横图 / 竖图 / 方图差异化导出
- 页面截图式导出
- 复杂设置中心、头像或账号资料
