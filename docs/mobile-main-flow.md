# skymap-harmony v0.2 手机端主流程

`skymap-harmony` v0.2 的目标是让鸿蒙端从“能演示”进入“日常可用”。手机端只服务一条稳定链路：

```text
首页 -> 开始复盘 -> 选择照片 -> 写复盘 -> 保存进入阅读 -> 复制复盘数据 / 导出图片 / 导出复盘文件 / 上传家庭存储 -> 回首页刷新 -> 复盘库回看历史
```

## 当前主链路

1. 首页点击「开始复盘」。
2. 通过系统相册选择一张照片。
3. 编辑页填写标题与复盘字段。
4. 点击「保存并进入阅读」，记录写入本地历史。
5. 阅读页展示复盘记录，可编辑、复制复盘数据、导出图片、导出复盘文件、上传家庭存储。
6. 回到首页时重新加载本地历史，概览、最近一次复盘、复盘库同步刷新。
7. 进入「复盘库」查看历史记录，列表支持搜索、筛选和左滑删除。

## 复盘字段

手机端 v0.2 保留固定复盘结构，不新增字段名：

- 标题
- 视觉落点
- 落点原因
- 视线路径
- 画面事实
- 核心关系
- 是否成立
- 延伸理解
- 当前卡点

空字段在阅读页和导出图中不展示；复制 JSON 时仍保留 key，值为空字符串。

## 首页

首页固定结构：

1. 摄影复盘卡
2. 开始复盘按钮
3. 复盘概览
4. 最近一次复盘
5. 复盘库

首页不展示完整记录列表，不展示横图 / 竖图 / 尺寸信息。

## 首页统计口径

首页所有模块都从 `ReviewCardHistoryService.load` 返回的历史记录生成：

- 复盘概览统计全项目记录。
- 最近一次复盘取最新一条历史记录。
- 复盘库使用默认项目 `default` 的同一份历史记录。
- 当前只有 `default` 项目时，首页总数等于默认项目全部记录数。
- 成立、待判断、不成立统一通过 `normalizeReviewJudgement` 归一化。
- `成立 + 待判断 + 不成立 = 总复盘`。

保存、导出标记和删除后，相关页面重新读取或重建这份历史数据；首页每次显示时都会重新加载。

## 阅读页

阅读页是「复盘记录详情」，不是导出编辑页。

固定内容：

- 顶部标题：复盘记录
- 操作入口：编辑、复制复盘数据、导出图片、导出复盘文件、上传家庭存储
- 正文：照片、标题、复盘时间、成立状态、已填写字段

阅读页不显示：

- 已保存 / 已导出这类状态正文
- 已复制复盘数据正文
- 横图 / 竖图 / 尺寸信息
- 导出按钮或系统状态栏出现在导出图中

## 导出图片规则

导出目标是一张复盘长图，不是当前手机页面截图。

导出图包含：

- 照片
- 标题
- 复盘时间，位于标题下方左侧
- 成立状态，位于同一行右侧
- 视觉落点
- 落点原因
- 视线路径
- 画面事实
- 核心关系
- 是否成立
- 延伸理解
- 当前卡点

空字段不导出，不留空白。导出图不包含状态栏、返回、编辑、复制复盘数据、导出图片、导出复盘文件、已保存、Toast 文案。v0.2 继续使用统一长图结构，不重新区分横图、竖图、方图导出版式。

导出时页面会临时挂载隐藏的导出组件并对组件快照，不对阅读页截图；系统保存弹窗出现时，背景仍保持正常阅读页。

## 复制 JSON 协议

阅读页「复制复盘数据」复制 `ReviewCardExchangeSchema v1` 的 pretty print JSON；「导出复盘文件」使用同一份 schema 保存 `*.review.json`。协议字段见 [review-card-exchange-schema.md](./review-card-exchange-schema.md)。

关键约束：

- `reviewStructure` 固定为 `quickReview`。
- `decision` 输出英文枚举：`works`、`uncertain`、`notWorks`。
- `reviewerText` 来自复盘设置。
- 旧记录缺字段时不丢 key。
- 页面只显示 Toast：已复制复盘数据。

## 复盘设置

复盘设置包含复盘人和家庭存储配置。

- 首页提供「复盘设置」入口。
- 设置页填写复盘人名称。
- 设置页可选择家庭存储模式：`WebDAV` 或 `SMB`。
- WebDAV 模式填写地址、用户名、密码或 Token、远程目录，阅读页「上传家庭存储」会直接上传 `*.review.json`。
- SMB 模式填写地址或 IP、端口、共享名、远程目录、用户名、密码和可选工作组或域，阅读页「上传家庭存储」会直接连接 SMB 共享并写入 `*.review.json`。
- 使用本地 preferences 持久化。
- 未设置时 `reviewerText` 输出空字符串。
- 复制 JSON 时读取当前保存的复盘人。

## 复盘库

复盘库是长期回看页：

- 不提供显式返回按钮，依赖系统左滑返回。
- 不显示「添加复盘」大按钮。
- 支持搜索标题、画面事实、核心关系、延伸理解和当前卡点。
- 支持按是否成立、当前卡点和复盘人筛选。
- 旧记录缺少复盘人时仍归入「全部复盘人」，但不进入具体复盘人筛选项。
- 列表支持左滑删除。
- 左滑删除只删除本地历史记录，不删除原照片、导出图片或已导出的 `review.json`。
- 列表滚动到底部不回弹。
- 卡片缩略图统一 1:1 cover。
- 摘要优先级：核心关系、当前卡点、延伸理解、还没有提炼核心关系。
- 状态角标位于卡片右上角并弱化显示。

## 当前不做

v0.2 不做：

- 多项目创建
- 云同步
- 账号登录
- AI 分析
- Mac 端改动
- icon 改动
- 字段名称重做
- 横图 / 竖图 / 方图差异化导出
- 页面截图式导出
- 复杂设置中心、头像或账号资料

## 命令行验证

以下脚本不依赖真机，可在仓库根目录直接运行：

```bash
node scripts/verify_review_logic.mjs
node scripts/verify_home_stats.mjs
node scripts/verify_image_layout.mjs
```

覆盖点包括：是否成立归一化、`decision` 映射、复制 JSON 字段完整性、空字段保留 key、首页统计、摘要 fallback、横图 / 竖图 / 方图方向判断和旧数据兼容。
