# skymap-harmony 手机端 V1 主流程基准

## 产品定位

`skymap-harmony` 是手机端摄影复盘工具，不是 Mac 端复刻、宣传页或参数面板。

V1 只服务一条主链路：

```text
选择照片 -> 写复盘 -> 看复盘卡 -> 保存或导出
```

所有页面都围绕快速复盘降噪。后续迭代如果影响这条链路的清晰度，默认不进入 V1。

## 设计原则

- 手机端只做快速复盘。
- 文案少，按钮语义清楚。
- 首页是工具入口，不是营销首页。
- 编辑页是写作页，不展示大段说明。
- 预览页只用于确认、编辑、保存和导出。
- 复盘卡本体优先，照片和文字必须像同一篇复盘笔记。
- 用户输入的长文本优先完整展示，不为固定高度裁切内容。
- 调试信息、技术兜底信息和开发解释不大面积展示给用户。

## 首页

首页目标：用户立刻知道可以选择照片开始复盘，同时区分长期归档和最近快捷入口。

固定结构：

1. 顶部标题区
   - 标题：摄影复盘卡
   - 副标题：选择照片，写下观察，生成一张复盘卡
2. 主按钮
   - 文案：选择照片，开始复盘
   - 首页只保留这一个主按钮
3. 我的复盘
   - 默认项目：我的摄影复盘
   - `projectId`: `default`
   - 作为长期归档入口，点击进入项目详情页
4. 示例入口
   - 标题：示例
   - 入口：横图 / 竖图 / 方图
   - 示例入口弱化，不写大段解释
5. 最近记录
   - 标题：最近记录
   - 只展示最近 3 条
   - 作为最近编辑或查看的快捷入口，点击直接进入阅读页
   - 无记录：暂无复盘记录
   - 有记录：展示缩略图、标题、时间

首页不展示：

- 营销化大标题
- 「真实照片 / 自动版式 / 可导出」标签
- 示例大卡片
- 最近记录里的长数字、导出状态和过多状态标签

## 编辑页

编辑页目标：让用户安心写复盘。

固定结构：

1. 顶部导航
   - 左侧：返回
   - 标题：写复盘
2. 顶部照片区
   - 照片位于写作内容顶部，是页面第一优先级
   - 宽度控制在内容区 90% 到 95%，高度按原图比例自适应
   - 图片按原比例 `Contain` 显示，不拉伸、不裁切关键主体
   - 竖图过高时允许页面整体滚动，不压缩成缩略图
   - 不显示横图 / 竖图 / 尺寸等技术信息
3. 写作区
   - 标题
   - 视觉落点
   - 落点原因
   - 视线路径
   - 画面事实
   - 核心关系
   - 是否成立（下拉选择：成立 / 不确定 / 不成立）
   - 延伸理解
   - 当前卡点
   - 默认标题：这张照片是否成立
4. 底部主按钮
   - 文案：保存并进入阅读
   - 键盘弹出时可以隐藏，避免遮挡输入框

交互要求：

- 输入区高度规范：
  - 标题：1 行
  - 视觉落点：2 行
  - 落点原因：2 行
  - 视线路径：3 行
  - 画面事实：3 行
  - 核心关系：2 行
  - 是否成立：下拉选择框
  - 延伸理解：2 行
  - 当前卡点：2 行
- 默认按以上行数展示。
- 内容超出后自动增长，不出现内部滚动条。
- 不允许固定超大输入框。
- 不允许所有字段高度一致。
- 字段视觉权重分两级：视觉落点、落点原因、视线路径、画面事实、核心关系使用 16 号标题和 16 号输入；是否成立、延伸理解、当前卡点使用 14 号标题和 15 号输入，其中是否成立使用固定三项下拉选择，避免异常值。
- 空间不足时优先压缩输入框，不压缩照片。
- 当前输入框在键盘弹出后必须可见。
- 当前卡点不能被键盘和底部按钮遮挡。
- 长文本输入不能丢失。

编辑页不展示：

- 长段说明
- 方向判断技术解释大卡片
- 模板参数、字号滑杆、颜色选择

## 预览页

预览页目标：确认复盘卡，并执行明确操作。

固定结构：

1. 顶部
   - 标题：复盘卡预览
   - 副标题：确认后可以保存或导出
   - 右侧按钮：回首页
2. 轻状态
   - 未保存
   - 已保存
   - 已导出
3. 复盘卡预览
   - 展示复盘卡本体
   - 预览和导出共用同一个组件
4. 底部操作
   - 编辑
   - 保存
   - 导出

预览页不展示：

- 含义不明的「完成」
- 过度突出的保存状态
- 大面积导出说明
- 复杂参数调节

## 复盘卡本体

V1 统一使用纵向阅读卡。横图、竖图、方图都先走同一结构，不再做左图右文、三栏或字段小卡片并排。

固定结构：

```text
照片
标题
视觉落点
正文
落点原因
正文
视线路径
正文
画面事实
正文
核心关系
正文
是否成立
正文
延伸理解
正文
当前卡点
正文
```

当前复盘框架：

```text
视觉落点 -> 落点原因 -> 视线路径 -> 画面事实 -> 核心关系 -> 是否成立 -> 延伸理解 -> 当前卡点
```

视觉要求：

- 字段纵向排列。
- 用户写多少，优先展示多少。
- 长文本自然换行。
- 长数字、英文和无空格字符串不能撑爆卡片。
- 分割线轻，不抢内容。
- 字段名浅灰、小号。
- 字段内容深色、清晰。
- 照片、标题、字段内容左边缘对齐。
- 卡片像照片复盘笔记，不像表单摘要。

## 保存记录

保留本地最近记录能力，并默认归档到「我的摄影复盘」项目。

- 编辑页生成预览时保存当前复盘。
- 预览页点击「保存」更新记录。
- 首页最近记录可再次打开。
- 首页「我的摄影复盘」进入项目详情页，作为长期回看列表。
- 最近记录和项目详情里的单条记录右上角展示是否成立角标，状态固定为「成立」「不确定」「不成立」。
- 保存记录时默认写入 `projectId: default`。
- 编辑后再次保存应更新原记录，不无限重复。

## 导出图片

导出目标是一张复盘卡图片，不是整个手机屏幕。

要求：

- V1 统一导出竖向长图，不再按横图 / 竖图 / 方图分版式。
- 导出结构固定为照片、标题、八个复盘字段。
- 导出卡能容纳较长文字。
- 文字过长时卡片高度可以增长。
- 导出图片可以是长图。
- 不为了固定高度裁掉用户输入。
- 空字段不导出，字段间距和行距保持紧凑但可读。
- 导出不包含返回、编辑、导出、保存状态、手机状态栏、页面标题和底部按钮。
- 导出成功或失败给轻提示。
- 导出按钮防止重复触发。
- 系统权限或保存确认不连续打扰用户；用户拒绝后不在同一次操作中反复弹出。
- 横竖图差异化导出保留为后续能力。

## V1 验收

1. 首页像工具首页，不像宣传页。
2. 编辑页像写作页，不像说明书。
3. 预览页按钮语义清楚。
4. 复盘卡统一为纵向阅读结构。
5. 长文本预览和导出不裁切。
6. 导出不会重复触发权限或保存流程。
7. 最近记录仍可用。
