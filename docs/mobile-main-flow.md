# skymap-harmony 手机端主流程

本文描述当前 HarmonyOS 端真实主链路。历史阶段清单、Beta 冻结记录和阶段性回归审计不再作为当前事实来源；当前事实以代码、README、产品层契约和本文件为准。

## 当前主链路

```text
首页 -> 开始复盘 -> 选择照片 -> 写复盘 -> 保存进入阅读 -> 导出菜单 -> 回首页刷新 -> 复盘库回看历史
```

1. 首页点击「开始复盘」。
2. 通过系统相册选择一张真实照片；读取宽高失败时保留 URI，并使用兜底比例继续进入编辑。
3. 编辑页填写标题与复盘字段。
4. 点击「保存并进入阅读」，记录写入本地 RDB 复盘库主索引，同时写入 `review_exchange` 沙箱备份。
5. 阅读页展示复盘记录，支持编辑、导出菜单和回看。
6. 导出菜单提供：
   - 导出复盘包：包含 `review.json`、导出图、缩略图和 `manifest.json`，用于家庭存储和 Mac 接力。
   - 导出复盘包（含原图）：包含原始照片和复盘数据，适合在 Mac 端继续处理。
   - 导出 `review.json`：只导出复盘数据文件，不包含图片。
   - 复制复盘数据：复制当前复盘的文本 JSON。
7. 回到首页时重新加载本地历史，概览、最近一次复盘和项目统计同步刷新。
8. 进入复盘库查看历史记录，列表支持搜索、状态筛选和删除。

## 复盘字段

手机端保留固定复盘结构，不新增字段名：

- 标题
- 视觉落点
- 落点原因
- 视线路径
- 画面事实
- 核心关系
- 是否成立
- 延伸理解
- 当前卡点

空字段在阅读页和导出图中不展示；复制或导出的 `review.json` 仍保留 key，值为空字符串。

## 首页与统计

首页聚焦快速进入复盘和查看状态，不承载完整记录列表。

首页所有模块都从 `ReviewCardHistoryService.load` 返回的历史记录生成：

- 复盘概览统计全项目记录。
- 最近一次复盘取最新一条历史记录。
- 当前默认项目使用同一份历史记录。
- 成立、待判断、不成立统一通过 `normalizeReviewJudgement` 归一化。
- `成立 + 待判断 + 不成立 = 总复盘`。
- 保存、导出标记和删除后，相关页面重新读取或重建历史数据。
- 历史读取失败时保留上一轮已显示统计，并提示用户重新进入首页或重启应用。

## 阅读页

阅读页是「复盘记录详情」，不是导出编辑页。

固定内容：

- 顶部标题：复盘记录
- 操作入口：编辑、导出菜单
- 正文：照片、标题、复盘时间、成立状态、已填写字段

阅读页不显示：

- 已保存 / 已导出这类状态正文
- 已复制复盘数据正文
- 横图 / 竖图 / 尺寸信息
- 导出按钮或系统状态栏出现在导出图中

## 导出图片规则

导出目标是一张复盘长图，不是当前手机页面截图。

导出图包含：

- 照片
- 标题
- 复盘时间，位于标题下方左侧
- 成立状态，位于同一行右侧
- 视觉落点
- 落点原因
- 视线路径
- 画面事实
- 核心关系
- 是否成立
- 延伸理解
- 当前卡点

空字段不导出，不留空白。导出图不包含状态栏、返回、编辑、复制复盘数据、导出按钮、已保存、Toast 文案。当前导出由隐藏导出组件生成截图；横图使用专门横向信息面板，竖图 / 方图使用默认单列长图结构。

## review.json 手动交换

阅读页「导出 review.json」会先写入沙箱备份，再通过系统文件保存弹窗把文件交给用户选择的位置。

必须保持以下反馈边界：

- 用户完成系统保存后，提示导出成功，并把 `exportedPath` 写回 RDB。
- 用户取消系统保存时，不伪装成导出成功；沙箱备份可保留用于恢复。
- 导出失败时提示用户稍后重试。
- `review.json` 只表达复盘数据，不包含 bundle 路径、导出图、缩略图或原图二进制。

协议字段见 [review-card-exchange-schema.md](./review-card-exchange-schema.md)，产品语义见 [REVIEW_JSON_SEMANTICS.md](./product/REVIEW_JSON_SEMANTICS.md)。

## 复盘包与家庭存储

导出复盘包是当前跨端接力的主路径。

- v1 复盘包包含 `review.json`、成品复盘图、缩略图、`manifest.json` 和 assets 说明。
- v2 原图复盘包包含原始照片和复盘数据，用于 Mac 端继续处理。
- 复盘包上传到家庭存储 `Skymap/ReviewBundles/...`。
- 家庭存储未配置、配置不完整、不可达、认证失败、目录创建失败、原图读取失败或上传失败时，必须给出用户可理解的失败反馈。
- 复盘包是外部导出资产，不参与 HarmonyOS 本地复盘库主索引查询。

详细契约见 [REVIEW_BUNDLE_V1_V2_CONTRACT.md](./product/REVIEW_BUNDLE_V1_V2_CONTRACT.md)。

## 本地存储

当前复盘库主索引是 RDB `reviews`。

- `ReviewCardStore` 只暂存当前编辑 / 阅读中的工作副本。
- `ReviewCardHistoryService` 是页面层唯一历史读写入口。
- 保存、更新、删除、导出标记主写 RDB。
- `review_exchange/*.review.json` 是沙箱备份、交换和有限恢复来源，不是复盘库主查询源。
- `Preferences(review_card_history.items)` 只作为旧版本迁移 / 诊断来源。
- 删除复盘记录会删除 RDB 索引项和对应沙箱备份，不删除原始照片、图库导出图、用户手动导出的 JSON 或家庭存储复盘包。

存储边界见 [REVIEW_LIBRARY_STORAGE_AUDIT.md](./product/REVIEW_LIBRARY_STORAGE_AUDIT.md)。

## 我的、设置与同步入口

- 我的页提供复盘人、首页图片、家庭存储、同步中心等入口。
- 复盘人名称影响新建复盘、复制 JSON、导出 `review.json` 和复盘包摘要。
- 家庭存储页负责 SMB 配置、保存、连接测试和状态反馈。
- 同步中心展示家庭存储配置状态和连接检查，不宣称自动同步、后台扫描、双向同步或冲突合并已经完成。

## 当前不做

- 不做账号系统、云同步或 AI 自动分析。
- 不做完整模板市场。
- 不做自动同步、批量导入、双向同步、远端删除或冲突合并。
- 不把家庭存储复盘包写回 `review.json` 字段。
- 不把原图二进制写入 HarmonyOS 本地复盘库。
