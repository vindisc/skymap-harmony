# skymap-harmony 手机端主流程基准

## 产品定位

`skymap-harmony` 是一个手机端「摄影复盘卡」工具，不是 Mac 端边框 App 的复刻。

它的核心价值是：用户选择一张照片，用很低成本写下第一眼观察、是否成立和卡点，然后生成一张清晰、干净、可保存、可导出的摄影复盘卡。

当前阶段只围绕这个闭环推进，不扩展模板市场、社区、登录、复杂参数和跨端能力。

## 手机端主链路

首页 -> 选择照片 -> 填写复盘 -> 预览复盘卡 -> 保存 / 导出 -> 回到首页记录

这条链路是后续所有迭代的判断基准：

- 首页只负责开始复盘和查看最近记录。
- 编辑页只负责写作，不负责复杂样式调节。
- 预览页只负责确认成品、保存记录和导出图片。

## 首页设计

首页目标是让用户第一眼知道：点一个按钮，选择照片，开始复盘。

首页结构：

1. 产品主视觉区
   - 标题：摄影复盘卡
   - 主文案：把一张照片整理成一次清晰复盘
   - 副文案：选择照片，写下第一眼、判断和卡点，生成一张复盘卡
2. 主按钮
   - 文案：选择照片，开始复盘
   - 这是首页唯一主动作，视觉权重最高
3. 示例体验区
   - 文案：不选照片也能试
   - 提供横图 / 竖图 / 方图三个辅助入口
   - 不能抢主按钮注意力
4. 最近记录区
   - 有记录时展示列表
   - 无记录时展示空状态
   - 卡片信息包含：缩略图、标题、方向、保存时间、导出状态
   - 点击记录进入预览页

首页不做：

- 复杂菜单
- 模板市场
- 参数调节
- 登录
- 社区

## 选择照片流程

目标是让用户选一张真实照片，并自动进入复盘编辑。

流程：

1. 点击「选择照片，开始复盘」
2. 打开系统相册
3. 用户选择一张照片
4. App 获取图片 URI
5. 尽量读取宽高
6. 自动判断方向：横图 / 竖图 / 方图
7. 创建 `ReviewCardDocument`
8. 进入编辑页

异常处理：

- 用户取消：回到首页，不报错
- 读取失败：提示用户，并保留示例入口
- 不能读取宽高：使用兜底尺寸继续流程，但界面必须诚实提示当前方向判断可能不准确

## 编辑页设计

编辑页目标是让用户像写短评一样写复盘，而不是填写复杂表单。

页面结构：

1. 顶部导航
   - 返回
   - 页面标题：写一张照片复盘
2. 照片信息卡
   - 左侧显示缩略图
   - 右侧显示方向、尺寸
   - 提示：照片方向会自动匹配复盘卡版式
3. 输入区
   - 标题
     - 默认：这张照片是否成立
   - 第一眼落点
     - 提示：我第一眼看到了什么？为什么？
   - 是否成立
     - 提示：成立 / 不确定 / 不成立，并说明原因
   - 卡点
     - 提示：这张照片当前最大的问题是什么？
4. 底部操作
   - 主按钮：生成复盘卡

交互要求：

- 输入框必须适合长文本
- 键盘弹出时，当前输入框必须可见
- 卡点输入不能被键盘遮挡
- 底部按钮不能遮挡输入
- 返回后内容不能丢
- 允许先写很短，后续再补

编辑页不做：

- 复杂模板参数
- 字号滑杆
- 颜色选择
- 多模板切换

## 预览页设计

预览页目标是让用户确认这张复盘卡是否可以保存或导出。

页面结构：

1. 顶部区域
   - 状态：已自动保存 / 未保存
   - 标题：复盘卡预览
   - 副文案：确认效果后，可以继续编辑、保存记录或导出图片
   - 次按钮：编辑
2. 复盘卡预览区
   - 居中展示完整复盘卡
   - 预览效果尽量接近导出效果
   - 卡片整体不能像调试组件
   - 照片、标题、文字必须属于同一张统一作品卡
3. 底部操作区
   - 编辑
   - 保存
   - 导出图片

预览页不做：

- 复杂参数调节
- 滤镜
- 裁切
- 社区分享

## 复盘卡版式原则

复盘卡不是「照片 + 表单摘要」。

复盘卡是一张完整的摄影学习记录卡。

结构关系：

- 照片是主视觉
- 标题是判断
- 第一眼落点 / 是否成立 / 卡点是判断依据

横图：

- 上图下文
- 照片宽度与文字区对齐
- 标题紧跟照片
- 信息区像一组复盘说明，不像三个输入框
- 中文不要被硬断行

竖图：

- 手机端优先保证可读性
- 如果左图右文导致文字过窄，手机端使用上图下文
- 不为了形式上的方向适配牺牲成品效果

方图：

- 上图下文
- 照片和文字比例平衡
- 信息区紧凑清晰

统一视觉：

- 白底
- 浅灰信息区
- 黑色主文字
- 灰色字段名
- 圆角克制
- 留白统一
- 不加复杂装饰
- 不做海报风

## 保存记录

目标是让用户回到首页后，能看到自己做过的复盘。

保存逻辑：

- 预览页自动保存或手动保存
- 保存 `ReviewCardDocument`
- 首页最近记录展示
- 点击记录可以再次查看
- 编辑后再次保存应更新原记录，不无限重复

## 导出图片

目标是让用户得到一张可以保存的复盘卡图片。

要求：

- 导出的是复盘卡本身，不是整个手机屏幕
- 导出成功有提示
- 导出失败有原因
- 如果当前阶段不能高清导出，要诚实标注限制，但按钮不能是假功能

## 优先级

P0：

1. 主链路完整
2. 编辑页键盘可用
3. 复盘卡照片和文字关系成立
4. 最近记录可用
5. 导出按钮不骗人

P1：

1. 首页产品感
2. 预览页视觉统一
3. 横图 / 竖图 / 方图体验优化
4. 保存状态反馈

P2：

1. 更多模板
2. AI 辅助
3. Mac 端 `photoreview` 互通
4. 云同步
5. 社区分享
