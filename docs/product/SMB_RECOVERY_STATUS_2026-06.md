# SMB Recovery Status 2026-06

日期：2026-06-13

## 1. 恢复来源

- 恢复目标：Harmony 当前主线的 SMB 家庭存储最小能力
- 历史完整可恢复点：`634edb9`
- 直接实现 SMB 直连上传的功能提交：`a90cea7`
- 本轮采用方式：按最小恢复原则手工摘取历史 SMB 代码，不整支 merge，不恢复 WebDAV，不恢复 Sync v1 bundle

## 2. 恢复范围

本轮只恢复以下能力：

- 家庭存储页配置 SMB 地址或 IP
- 家庭存储页配置共享目录
- 家庭存储页配置用户名
- 家庭存储页配置密码或凭据
- 家庭存储页配置目标路径
- 阅读页把当前复盘导出的 `review.json` 上传到家庭存储
- 保留现有“导出 `review.json` 到系统文件”链路，与 SMB 上传并存

## 3. 当前支持能力

当前 Harmony 代码基线已恢复：

- 家庭存储页负责 SMB 设置保存、读取与连接测试；设置页只保留进入家庭存储页的入口
- 家庭存储密码安全存储
- SMB 连接测试
- 端口数字输入、密码显隐和保存/测试结果反馈
- 当前复盘 `review.json` 上传到 SMB 家庭存储
- 上传成功提示
- 上传失败提示
- 阅读页上传入口

当前恢复后的最小链路是：

```text
Harmony
↓
上传 review.json 到家庭存储（SMB）
```

## 4. 当前不支持能力

本轮明确不支持：

- WebDAV 上传
- Mac 自动发现 SMB 目录
- Mac 自动导入
- 双向同步
- 冲突合并
- 照片同步
- Review Bundle
- Manifest
- SMB / WebDAV 自动同步

## 5. 真机回归清单

当前代码已恢复，但 Beta 状态仍取决于真机回归结果。建议至少执行：

1. 配置 SMB 地址
2. 配置共享目录
3. 配置用户名/密码
4. 配置目标路径
5. 从设置页进入家庭存储页，执行“测试连接”
6. 在阅读页执行“上传到家庭存储”
7. 确认 NAS / 家庭存储中出现 `review.json`
8. 配错账号或路径后确认有明确失败提示
9. 再执行一次“导出 review.json”到系统文件，确认旧链路仍可用

## 6. 已知风险

- `Smb2Client.ets` 是历史低层协议实现，仍需依赖真实设备和真实 NAS 环境验证兼容性
- 当前恢复的是最小上传能力，不代表已经进入稳定同步系统
- 不同 NAS 的 SMB 权限、目录结构和认证策略可能导致差异
- Product Layer 口径应表述为：SMB 代码已恢复到当前基线，但仍需真机验收后才能决定 Beta 对外承诺
