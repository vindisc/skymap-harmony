# Review Bundle 家庭存储浏览

日期：2026-06-28

本文定义 Mac 端“家庭存储复盘包浏览”能力。本能力只负责浏览家庭存储中的 v1 / v2 review bundle，并由用户手动选择单个复盘包导入本机复盘库。

## 一、功能定位

用户在家庭存储中已经有 HarmonyOS 导出的 `ReviewBundles` 目录。Mac 端可以选择这个根目录，扫描其中已有复盘包，显示候选列表，并复用现有导入逻辑导入单个 bundle。

典型目录：

```text
Skymap/
└── ReviewBundles/
    ├── 2026/
    │   └── 06/
    │       ├── review_20260627_212801_xxxxxxxx/
    │       └── review_20260628_153000_xxxxxxxx/
    └── review_20260701_101500_xxxxxxxx/
```

## 二、当前能力

- 选择家庭存储中的 `ReviewBundles` 根目录。
- 记住最近选择的根目录，下次打开时可继续使用。
- 从根目录向下扫描最多三层。
- 支持 `ReviewBundles/YYYY/MM/review_xxx/`。
- 支持 `ReviewBundles/review_xxx/`。
- 读取每个候选 bundle 的协议文件并复用 Mac Reader 校验。
- 识别 v1 成品图复盘包与 v2 原图复盘包。
- 展示标题、类型、判断、时间、来源、导入状态和单个导入操作。
- 通过 `bundleId` 判断是否已导入。
- 已导入项可以定位本机复盘库记录。
- 未导入且校验通过的候选项可以单个导入。

## 三、v1 / v2 行为

- v1 成品图复盘包导入后进入本机复盘库，只读查看 `review-card.png`。
- v1 不显示“打开为复盘卡”。
- v1 不把成品图当作原图进入主编辑器。
- v2 原图复盘包导入后进入本机复盘库，可由用户显式打开为复盘卡。
- v2 打开为复盘卡时恢复原图、复盘内容和模板配置。
- v2 `exportedImages=[]` 合法，不要求成品图。

## 四、不包含

- 自动同步。
- 后台定时扫描。
- 批量导入。
- 双向同步。
- 远端删除。
- 写回家庭存储。
- 冲突自动合并。
- 云数据库。
- 修改 Review JSON 字段。
- 修改 manifest v1 / v2 协议。
- HarmonyOS 端实现变更。

## 五、权限与路径

Mac 使用系统目录选择器选择 `ReviewBundles` 根目录。客户端会优先保存 security-scoped bookmark，并保留普通路径作为降级记忆。若下次打开时目录不可访问，用户需要重新选择。

当前能力不假设家庭存储路径永久有效，也不扫描用户整个磁盘。

## 六、验收路径

1. 准备包含 v1 / v2 的 `ReviewBundles` 根目录。
2. 打开 Mac 复盘库。
3. 点击“浏览家庭存储复盘包”或切换到“家庭存储”。
4. 选择 `ReviewBundles` 根目录。
5. 预期扫描出 v1 / v2 候选列表。
6. v1 显示“成品图复盘包”。
7. v2 显示“原图复盘包”。
8. 已导入项显示“已导入”，并可查看本地记录。
9. 点击 v1 导入后，本机复盘库显示只读成品图。
10. 点击 v2 导入后，本机复盘库显示原图，并可打开为复盘卡。
11. 选择错误目录时提示没有找到复盘包或检查目录权限。
