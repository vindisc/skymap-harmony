# 首页图片配置 v1

## 方案选择

- 本轮不新建 RDB 表。
- 首页图片配置数据量小，不需要复杂查询，也不属于复盘主索引。
- v1 使用 Preferences 保存配置元数据，使用 `filesDir/home_hero/` 保存图片副本。

## 行为规则

- 未配置图片时，首页继续展示内置默认 Hero 图。
- 配置 1 张图片时，首页固定展示该图片。
- 配置多张图片时，首页按添加顺序自动轮播，默认间隔 4 秒。
- 用户从“我的 / 首页图片”添加、删除、清空图片。
- 删除全部图片后，首页回退默认图。

## v1 边界

- 不改 Review JSON。
- 不改 RDB 复盘库表结构。
- 不改 review bundle。
- 不接入家庭存储、云同步或远端同步。
- 不做图片裁剪、压缩、EXIF 读取或排序拖拽。

## 后续建议

- 如果用户上传超大原图，后续增加压缩副本，避免首页加载和存储压力。
- 如果后续做主题库、多端同步或配置历史，再评估迁移到 RDB。
