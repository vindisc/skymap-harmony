# Skymap Product Workflow

## 工作流原则

- 手机端和桌面端服务同一个摄影工作流，不各自发明闭环。
- review.json 是复盘信息的核心交换物，而不是某个客户端的临时剪贴板内容。
- 资料、模板、预设和导出设置是可复用资产，不是一次性页面参数。
- 任何新增客户端能力都必须说明它处于哪个产品工作流节点。

## 工作流 1：拍摄到复盘查看

```text
拍摄
↓
复盘
↓
review.json
↓
同步
↓
查看
```

### 目标

让用户在拍摄后快速记录一次观看判断，并能在另一个设备或之后的时间继续查看、整理和使用。

### 产品步骤

1. 用户完成拍摄。
2. 用户在 Harmony Client 或 Mac Client 创建复盘。
3. 复盘内容保存为 Review，并可输出 review.json。
4. review.json 通过手动文件、系统分享、家庭存储或未来同步系统流转。
5. 用户在复盘库中查看历史复盘，并能继续编辑、导出或归档。

### 客户端职责

| 节点 | Mac Client | Harmony Client |
| --- | --- | --- |
| 拍摄后快速记录 | 可支持，但不是主要入口 | 主要入口 |
| review.json 创建 | 需要支持完整归档和导入 | 需要支持轻量创建和导出 |
| 同步 | 规划接入家庭存储或文件夹导入 | 规划接入 SMB / WebDAV / 文件导出 |
| 查看 | 需要建设产品级复盘库 | 已有本地历史与默认项目 |

### 验收口径

- 复盘字段语义跨端一致。
- 空字段、旧字段和缺失字段有明确 fallback。
- 用户知道文件保存在哪里，失败时知道下一步怎么办。
- 查看页不是导出图截图，导出图也不是操作页截图。

## 工作流 2：导入照片到批量导出

```text
导入照片
↓
选模板
↓
调参数
↓
批量导出
```

### 目标

让用户把一组照片转成风格稳定、命名清晰、可交付的成品图片。

### 产品步骤

1. 用户导入单张或多张照片。
2. 系统读取照片尺寸、方向和可用 EXIF。
3. 用户选择模板。
4. 用户根据照片内容调整模板参数。
5. 用户复用资料中心和预设，减少重复输入。
6. 用户设置导出比例、质量、尺寸、命名规则和归档方式。
7. 用户执行单张或批量导出。
8. 系统返回成功位置、失败原因和可继续操作入口。

### 客户端职责

| 节点 | Mac Client | Harmony Client |
| --- | --- | --- |
| 多图导入 | 主要入口 | 当前不承担 |
| 模板选择 | 主要入口 | 仅复盘卡相关模式 |
| 参数调整 | 主要入口 | 轻量复盘字段编辑 |
| 批量导出 | 主要入口 | 当前不承担 |
| 系统相册保存 | 可按平台扩展 | 主要用于复盘长图 |

### 验收口径

- 横图、竖图、方图都不能被同一种布局粗暴套用。
- 预览和最终导出应保持同一产品语义。
- 批量导出必须有顺序、命名、失败反馈和归档策略。
- 导出设置属于 ExportJob，不应被误写成单张照片的永久复盘内容。

## 工作流 3：资料中心到预设复用

```text
资料中心
↓
预设
↓
复用
```

### 目标

让用户把个人署名、品牌信息、复盘人、模板参数和导出规格沉淀为可复用资产，减少每次导出和复盘的重复操作。

### 产品步骤

1. 用户维护个人资料、品牌资料、复盘人或当前照片资料。
2. 用户在模板或导出设置中引用这些资料。
3. 用户把稳定参数保存为预设。
4. 用户在后续照片、复盘或导出任务中复用预设。
5. 默认预设只影响新任务，不覆盖已经明确编辑过的历史内容。

### 客户端职责

| 节点 | Mac Client | Harmony Client |
| --- | --- | --- |
| 个人资料 | 已支持 | 规划中 |
| 品牌资料 | 已支持 | 规划中 |
| 复盘人 | 可由资料中心承接 | 已支持轻量设置 |
| 模板预设 | 已支持 | 规划中 |
| 导出预设 | 已支持 | 规划中 |

### 验收口径

- 资料中心不是账号系统。
- 预设不是新模板的替代品。
- 默认预设不能悄悄覆盖用户已经编辑过的照片或复盘。
- 资料字段在模板、导出命名和 review.json 中使用同一语义。

## 统一同步节点

后续任何同步系统都必须围绕以下对象说明边界：

| 对象 | 是否必须同步 | 说明 |
| --- | --- | --- |
| Review | 必须 | 摄影复盘的核心。 |
| Photo | 可选 | 初期可只同步引用和摘要，不同步照片二进制。 |
| Template | 必须有版本 | 至少同步模板标识和语义。 |
| Preset | 可分阶段 | 先解决本地复用，再解决跨端复用。 |
| Profile | 可分阶段 | 个人、品牌、复盘人应统一语义。 |
| ExportJob | 不必全量同步 | 更适合作为归档记录和结果索引。 |
