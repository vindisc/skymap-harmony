# Skymap Product Roadmap

本路线图只列用户价值，不列技术重构任务。客户端实现顺序必须服从产品层目标。

## v0.x：稳定化

目标：让现有复盘、导出和归档链路稳定可信。

| 方向 | 用户价值 |
| --- | --- |
| 复盘字段稳定 | 用户在两个客户端看到同一套复盘问题，不会因为换设备丢字段或变含义。 |
| review.json 收敛 | 用户可以把一次复盘作为文件保存、迁移、导入和交给 GPT 交流。 |
| 导出反馈稳定 | 用户知道导出成功在哪里，失败为什么失败，还能找到兜底文件。 |
| 复盘历史稳定 | 用户能回看最近复盘和默认复盘项目，不担心记录丢失。 |
| 模板与预设边界稳定 | 用户理解模板是表达结构，预设是个人复用，不被重复入口干扰。 |
| 产品文档先行 | 新能力先进入产品层，避免两个客户端各自漂移。 |

## v1.x：摄影工作流

目标：把“复盘”和“导出”连接成完整摄影工作流。

| 方向 | 用户价值 |
| --- | --- |
| 产品级复盘库 | 用户能按时间、状态、照片、模板和关键词回看复盘。 |
| 搜索与筛选 | 用户能找到“所有不成立的人像”“最近一个月的当前卡点”“某个镜头拍的复盘”。 |
| 家庭存储同步 | 用户能用 SMB / WebDAV 或文件夹完成手机与桌面的低成本接力。 |
| Mac 与 Harmony 职责清晰 | 用户知道手机端负责快速记录，桌面端负责精修导出，两端数据能接上。 |
| 资料中心统一 | 用户的署名、复盘人、品牌信息和导出命名能复用，不用到处重复填。 |
| 导出归档 | 用户能把成品图、review.json 和导出记录放在同一条可追溯链路里。 |

## v2.x：摄影成长系统

目标：让复盘库从记录工具变成摄影成长的反馈系统。

| 方向 | 用户价值 |
| --- | --- |
| 成长统计 | 用户能看到自己的成立率、常见卡点、题材变化和输出节奏。 |
| 复盘趋势 | 用户能发现哪些问题反复出现，哪些判断正在改善。 |
| 作品线索 | 用户能从复盘库里筛出值得继续发展的一组照片和主题。 |
| 练习计划 | 用户能根据历史卡点形成下一轮拍摄目标。 |
| 风格资产 | 用户能沉淀自己的模板、资料、预设和输出习惯。 |
| 人机协作 | GPT 或其他助手可以基于用户授权的复盘资料给建议，但不替用户下判断。 |

## 暂不进入路线图

- 社交平台、Feed、点赞和关注。
- 大而全云相册。
- 通用修图能力。
- 以技术栈迁移为目标的跨端重写。
- 为了堆数量而增加重复模板。
