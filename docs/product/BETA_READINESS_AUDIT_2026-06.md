# Skymap Beta Readiness Audit 2026-06

日期：2026-06-13

结论：**P0 已修复，当前可以继续 Beta 收口验证**

## 本轮范围

本轮只解决一件事：

```text
用户真的能完成一次 Review 交换
```

明确不做：

- Sync v1
- review bundle
- manifest
- SMB 自动同步
- WebDAV 自动同步
- 标签系统
- 收藏系统
- 成长统计

## 已修复的 P0

### P0-1 Harmony 导出后用户拿不到 review.json

状态：**已修复**

- Harmony 导出 `review.json` 时，改为拉起系统保存面板，把文件交到用户可见的系统文件位置。
- 成功提示不再要求用户理解应用沙箱路径，而是明确说明已经保存到系统文件，可直接发送给 Mac。
- 如果用户未完成系统保存，会明确提示“未完成导出到文件”，并要求重新选择保存位置，不伪装成成功。

### P0-2 Mac 导入链路不可发现

状态：**已修复**

- Mac 的 Review Library 内“导入 review.json”现在是独立能力，不再依赖先导入照片、先选中照片或先切模板。
- 导入成功后会明确提示“已写入 Review Library”。
- 导入结果支持：
  - 立即查看
  - 定位记录
  - 打开复盘库

## 当前产品判断

在 `Sync v0.5` 基线下，`Harmony -> review.json -> Mac Review Library` 这条最小交换链路已经从“技术上存在”提升为“普通用户可理解并可完成”。

这不代表完整同步系统已经完成，也不代表可以开始宣传：

- SMB / WebDAV
- 自动同步
- review bundle
- manifest
- 家庭存储自动发现

补充口径：

- 上述能力在当前 Beta 基线里都不能当成已交付能力。
- 这不等于 Harmony 历史上从未做过相关实现；历史分支曾出现 WebDAV 上传、SMB 上传/系统保存模式、设置页配置和阅读页上传入口。
- 但这些代码与入口当前不在 Beta 分支，且没有当前真机回归结论，因此本轮只能按“未纳入 Beta 主链路、不能对外承诺”处理。

## 剩余注意点

- Mac 当前“导入摄影复盘”仍然是把 JSON 应用到当前照片上的模板内能力，它和 Review Library 独立导入是两条不同链路。
- Harmony 当前修复的是“导出到系统文件”能力，不是“直接发给指定 Mac 设备”的自动传输。
- 如果后续要恢复家庭存储链路，必须先确认历史实现是否重新合线，再补真机回归；不能只依据旧文档宣称 SMB / WebDAV 已支持。
- 后续如果继续推进 Sync，需要在不破坏这条手动可用链路的前提下迭代。
