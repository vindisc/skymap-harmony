# 当前产品说明

本文只描述当前 `skymap-photoreview` 仓库已经落地的能力，不写未来路线图，不把 Mac 侧规划当作 Harmony 当前能力。

## 1. 项目定位

当前产品是一款 HarmonyOS 手机上的摄影复盘工具，目标是把“导入照片、形成判断、保存回看、持续积累”做成单机闭环。

当前不是：

- 云同步产品
- 双端协同产品
- 完整摄影边框模板产品
- AI 辅助分析产品

## 2. 当前信息架构

应用入口固定为四个主 Tab：

- 首页
- 复盘库
- 统计
- 我的

### 首页

首页承担“开始动作”和“查看最近状态”，不承载完整历史列表。

当前入口是并行双按钮：

- `导入照片，开始复盘`
  - 单选一张照片
  - 直接进入编辑页
- `导入待复盘`
  - 多选照片
  - 批量写入待复盘列表
  - 当前上限 30 张

首页同时展示：

- Hero 图片区域
- 最近复盘状态
- 默认项目摘要

### 复盘库

复盘库同时承接两类数据入口：

- 已保存复盘
- 待复盘照片

当前筛选固定为：

- 全部
- 待复盘
- 成立
- 待判断
- 不成立

其中：

- `待复盘` 视图展示 `PendingReviewPhoto`
- 其余状态视图展示已保存 `Review`

当前没有把待复盘照片混入已完成复盘列表模型。

### 统计

统计页正式命名已经冻结为：

- `学习进度`
- `复盘结果`

这页同时读取：

- `PendingReviewPhotoStore`
- `ReviewCardHistoryService`

但两者职责分离。

### 我的

当前提供以下设置入口：

- 复盘人
- 首页图片
- 卡片背景
- 显示与动效
- 删除星河效果
- 家庭存储
- 同步中心
- 备份全部复盘
- 从备份恢复

这些入口是配置入口，不代表相关能力都已经成为完整产品闭环。

## 3. 当前核心能力

### 复盘编辑

当前复盘字段固定为：

- 标题
- 视觉落点
- 落点原因
- 视线路径
- 画面事实
- 核心关系
- 是否成立
- 延伸理解
- 当前卡点

### 保存与回看

- 保存后写入本地 RDB `reviews`
- 同时写入 `review_exchange` 沙箱备份
- 已保存复盘可从复盘库再次打开

### Pending 与 Review 关系

- `Pending` 是待复盘照片任务
- `Review` 是已保存的复盘结果
- 从待复盘照片进入编辑并保存后，对应 `Pending` 会被标记为 `reviewed`
- 当前不把 `Pending` 直接当成 `Review`

### 导出

当前导出链路包括：

- 导出 JPG 复盘卡到相册
- 导出复盘包
- 导出含原图的复盘包

导出位置与结果反馈必须清晰：

- JPG：先写应用沙箱，再通过系统保存流程进入相册
- 复盘包：写入家庭存储配置目标

## 4. 当前统计口径

### 学习进度

- `待复盘 = pendingCount`
- `已完成 = 已保存复盘总数`
- `累计导入 = pendingCount + 已保存复盘总数`
- `完成率 = 已保存复盘总数 / 累计导入`

### 复盘结果

只统计已保存复盘，不统计待复盘照片。

- `成立`
- `待判断`
- `不成立`
- `累计复盘`

因此当前必须冻结以下边界：

- 不再使用 `学习概览 / 复盘质量`
- 不把 `Pending` 混进 `复盘结果`

## 5. 当前外部能力边界

### 家庭存储

当前已经有：

- 配置保存
- 连接测试
- 从复盘详情导出复盘包到目标目录

当前还没有：

- 自动同步
- 后台扫描
- 双向同步
- 冲突合并

### review.json

- 是交换和备份格式
- 不是本地复盘库主查询源
- 当前字段继续冻结
- 独立文件导出入口当前已暂时屏蔽

## 6. 当前不做

- 自动同步、双向同步、冲突合并
- 账号系统、云端服务
- AI 自动分析
- 把待复盘数据并入 Review 主模型
- 以未来 Roadmap 作为当前实现说明
