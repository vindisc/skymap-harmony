# HarmonyOS v0 发布基线

本文定义 HarmonyOS 端 v0 的发布基线：当前已经成立的能力、明确不包含的范围、存储边界、回归清单、已知风险和下一阶段路线。它用于发布前验收、后续回归和跨端接力设计，不新增功能，也不改变现有数据结构。

## v0 当前能力范围

- 首页导入：从首页进入系统相册，选择真实照片开始复盘。
- 复盘编辑：填写标题、视觉落点、落点原因、视线路径、画面事实、最强关系、是否成立、延伸理解、当前卡点等复盘字段。
- 保存复盘：保存后写入本地复盘库主索引，并保留应用内备份。
- 复盘库：查看已保存复盘，支持回看记录。
- 搜索 / 筛选：复盘库支持按关键词搜索和按判断状态筛选。
- 更新复盘：从已保存记录进入编辑，修改后更新同一条复盘记录。
- 删除复盘：删除本地复盘库记录和对应应用内备份。
- 横图 / 竖图 / 方图导出：根据图片比例使用对应导出呈现，避免单一布局硬套所有照片。
- 导出内容清洗：导出图不带 placeholder、重复状态、页面操作按钮和普通界面噪音。
- 统计页：基于真实复盘库数据展示总数、判断分布、最近 30 天和最近卡点等轻量反馈。
- 我的页：展示复盘人、家庭存储、RDB 诊断等入口状态。
- 家庭存储配置入口：提供家庭存储状态查看、设置保存和连接测试入口。
- RDB 主读主写：RDB 是当前复盘库主索引，保存、更新、删除、导出标记走 RDB 主链路。
- `review_exchange` 备份：保存和更新复盘时生成应用内 `review_exchange/*.review.json` 备份，可作为交换和有限恢复来源。
- `Preferences` 旧数据迁移：`Preferences.items` 仅作为旧版本数据迁移来源，不再是复盘库主索引。
- 保存 / 导出 / 删除 / 导入反馈：关键操作已有轻量 loading、禁用态、防重复点击和成功 / 失败提示基线。

## v0 明确不包含

- 多设备同步。
- 家庭存储自动同步。
- Mac / Harmony 双端接力。
- 摄影边框完整模板系统。
- 复杂趋势图表。
- AI 自动分析照片。
- 账号系统。
- 云端服务。

## 存储基线

- RDB 是复盘库当前主索引。
- `review_exchange` 是应用内备份 / 交换 / 有限恢复来源。
- `Preferences.items` 只作为旧版本迁移来源。
- 删除复盘只删除 RDB 记录和应用内 `review_exchange` 备份。
- 删除复盘不删除原图。
- 删除复盘不删除用户已导出图片。
- 删除复盘不删除用户导出的 `review.json`。
- 删除复盘不删除家庭存储远端文件。

## Review JSON 边界

- Review JSON 是交换格式 / 导出备份格式。
- Review JSON 不是复盘库主索引。
- 当前不改 Review JSON 字段。
- Mac 端和 HarmonyOS 后续接力要以 Review JSON / review bundle 为边界，而不是把某端内部存储结构当作跨端协议。

## 已知风险

- 保存失败 UI 提示仍需加强。
- 导出失败 UI 提示仍需加强。
- 家庭存储目前只是配置入口，不是完整同步。
- `review_exchange` 是有限恢复，不是完整同步机制。
- 统计页是 v1 轻量反馈，不是完整成长系统。
- RDB 写失败需要更明确用户提示。
- RDB 开发诊断入口属于开发期能力，普通用户路径默认隐藏。

## 下一阶段路线

### 阶段 A：交互体感优化

- 保存中 / 导出中 / 删除中状态。
- 防重复点击。
- 失败提示。
- 删除确认优化。
- 空状态优化。
- 保存失败 / 导出失败 UI 提示继续做更细的场景校准。

### 阶段 B：review bundle v1

- `review.json`。
- 导出图片。
- `manifest.json`。
- 缩略图。
- 家庭存储目录规范。

### 阶段 C：Mac / Harmony 接力

- Harmony 导出 review bundle。
- Mac 导入 review bundle。
- 字段兼容。
- 冲突策略。

### 阶段 D：摄影边框能力

- 模板系统。
- 导出样式。
- EXIF / 署名 / 边框参数。

## 发布判定

v0 发布基线的判断标准是：本机单端复盘主流程可创建、保存、回看、更新、删除、导出和统计；RDB 承担复盘库主读主写；`review_exchange` 和 Review JSON 只承担备份、交换和有限恢复边界；所有未进入 v0 的同步、账号、AI、模板市场和完整摄影边框能力都留到后续阶段。
